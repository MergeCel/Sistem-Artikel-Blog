3. Kenapa Memilih NoSQL (MongoDB)
MongoDB dipilih karena; artikel blog dapat memiliki field berbeda tanpa harus mengubah struktur tabel, data disimpan dalam bentuk JSON/document sehingga cocok untuk aplikasi web modern, mongoDB cocok untuk aplikasi dengan data besar dan berkembang, tidak perlu membuat relasi tabel yang rumit seperti SQL, sistem artikel/blog lebih cocok menggunakan document database.
